Screen 3b — Permission Editor (YIQ — granular)
Configure: N&J Broker
Entity-level · action-level · field-level · data scope
N&J Broker Brand-scoped role
How to read this: Click an entity row to expand it and see action-level permissions. Green chips = enabled. Click any chip to toggle. Each action can also carry a Data Scope — set that in the Data Scope tab.
⚡
Data Scope is independent of the action permission. Even if this role can "delete", the scope controls how much they can delete — only their own content (Authored), their brand's content (Brand), all brands (Cross-Brand), or everything (All).
💡 Design note — scope ladder
The 4 scopes form a clear ladder from most restrictive to most open. A segmented control communicates this spectrum better than a dropdown.
Field-level control (YIQ only): For each entity, configure whether a field is hidden, visible read-only, or fully editable. This mirrors exactly what the user sees in the UI — no field is shown that the role cannot access.
